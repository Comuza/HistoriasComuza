1. ¿Qué es HTML?

HTML (HyperText Markup Language) es el lenguaje estándar para crear páginas web. Este lenguaje no es considerado un lenguaje de programación, sino un lenguaje de etiquetas, aunque esto no le quita importancia al lenguaje.

Este lenguaje no se maneja solo en la creación de páginas web, sino que va acompañado de otros 2 Lenguajes, CSS y JavaScript.

Este es el orden de cada uno:

HTML son los huesos

Con este le das estructura a la página, el esqueleto donde todo se sostiene.

CSS es la piel

Para que se vea divina la página, con colores, formas y diseño moderno.

JavaScript son los músculos

Le da la fuerza suficiente para crear un mundo casi ilimitado: botones, cambios de idioma, cambios de temas, manejar cookies, etc.

Esto es así porque, aunque HTML sea importante para la estructura de la web, sin los demás no sería posible crear sitios web modernos con botones, cambios de colores, idiomas y un diseño moderno con animaciones.

2. El Estándar Vivo y la Evolución

HTML no siempre fue como lo conocemos. Para entender el modelo actual "HTML5", hay que comprender que este lenguaje se creo en un mundo de caos donde cada navegador interpretaba el código a su manera.

El Pasado (HTML 4 / XHTML)

Este lenguaje era rígido, dependía de plugins externos como Flash para video y carecía de etiquetas estructurales modernas.

El Presente (HTML5)

Este lenguaje es el Estandar del mundo entero. Este introdujo multimedia nativa, geolocalización y la base de la web moderna: la Semántica.

HTML5 es el acuerdo global que permite que una web funcione igual en cualquier dispositivo, desde un smartphone hasta una Smart TV.

3. El "Boilerplate": La Estructura Base

Todo proyecto profesional nace de este esquema. Es el contrato legal entre tu código y el navegador:

<!DOCTYPE html>
        <html lang="es">
        <head>
        <meta charset="UTF-8"> <
        <meta name="viewport" content="width=device-width">
        <title>Mi Web Profesional</title>
        </head>
        <body>
        </body>
        </html>
Estructura básica

Ahora debemos entender porque funciona eso:

  • <!DOCTYPE html> - Declara que es HTML5, el modelo más moderno, y el estandar actual.
  • <html> - Aqui inicia el documento html
  • <head> - Aqui empieza el head, aqui se introduce todo lo que no es visible en la pagina, como el UFT-8, idioma de la pagina, etc.
  • <lang="es"> - Atributo que indica el idioma de la página, este indica que el idioma de la pagina, para facilitar accesibilidad, traducciones y cambios de idioma, <es> es español y <en> es inglés.
  • <meta charset="UTF-8"> - Define la codificación de caracteres, UTF-8 es el estándar que soporta casi todos los caracteres del mundo, incluyendo emojis, ñ, y una infinidad de caracteres más.
  • <meta name="viewport" content="width=device-width"> - Hace que la página sea responsiva, adaptándose a diferentes tamaños de pantalla, como móviles, tablets y computadoras.
  • <title> - El título que aparece en la pestaña del navegador, es importante para que los usuarios identifiquen tu página entre muchas pestañas abiertas.
  • </head> - Aqui termina el head, siempre para cerrar algo como head, debe empezar con /, sino se abre otra.
  • <body> - En el body se encuentra el contenido visible de la página, animaciones, colores y palabras
  • <title> - Título que aparece en la pestaña
  • </body> - Cierra el body
  • </html> - Cierra el documento html
Felicidades: Ya entendiste en que se basa cada pagina web profesional!, adentro de cada bloque como head y body se hace lo verdaderamente potente.
Etiquetas de texto

Las etiquetas de texto son las más básicas pero esenciales. Con ellas escribes todo el contenido que el usuario va a leer:

Etiqueta Qué hace Ejemplo
<h1> a <h6> Cada h (Heading) es un tipo de titulo, subtitulo etc... h1 es el más grande, y cada vez más grande el numero, más chico se vuelve el titulo, h6 es el menos importante. <h1>Mi título principal</h1>
<p> Párrafo de texto. El de siempre <p>Este es un párrafo</p>
<strong> Texto en negrita. Esto cambia la tipografia de un texto a negrita. <strong>Texto importante</strong>
<em> Texto en cursiva. Estp cambia la tipografia de un texto a cursiva. <em>Texto con énfasis</em>
<br> Salto de línea. No necesita cierre, solo hace un enter. Línea 1<br>Línea 2
<hr> Línea horizontal. Separa secciones visualmente. <hr>
💡 Dato clave: Se pueden combinar etiquetas para crear efectos más complejos, por ejemplo <h3>hola<em>mundo</em></h3> .
Enlaces e imágenes

Los enlaces son lo que hace que la web sea una "red". Conectan páginas entre sí. Las imágenes hacen todo más visual.

<!-- Enlace a otra página -->
<a href="https://ejemplo.com">Texto que se ve</a>

<!-- Enlace que abre en nueva pestaña -->
<a href="pagina.html" target="_blank">Abrir en nueva pestaña</a>

<!-- Imagen -->
<img src="imagen.jpg" alt="Descripción de la imagen">

Explicación línea por línea:

  • <a> - La etiqueta de ancla (anchor). Todo lo que esté adentro se vuelve clickeable.
  • href - El atributo que dice a dónde va el enlace. Puede ser otra web, otra página tuya, o incluso una sección dentro de la misma página.
  • target="_blank" - Hace que el enlace se abra en una pestaña nueva. Si no pones esto, se abre en la misma pestaña.
  • <img> - La etiqueta de imagen. No tiene cierre. Necesita dos cosas: src (de dónde viene la imagen) y alt (texto alternativo).
  • alt - Es súper importante. Si la imagen no carga, se ve este texto. También lo usan los lectores de pantalla para personas con discapacidad visual.
⚠️ Importante: El atributo alt en imágenes NO es opcional. Es obligatorio para accesibilidad web. Describe qué hay en la imagen.
Listas

Las listas sirven para organizar información en viñetas o números. Hay dos tipos principales:

<!-- Lista desordenada (con viñetas) -->
<ul>
  <li>Manzanas</li>
  <li>Peras</li>
  <li>Plátanos</li>
</ul>

<!-- Lista ordenada (numerada) -->
<ol>
  <li>Primero, precalentar el horno</li>
  <li>Segundo, mezclar los ingredientes</li>
  <li>Tercero, hornear por 30 minutos</li>
</ol>

Cuándo usar cada una:

  • <ul> (unordered list) - Usa viñetas. Perfecta para listas donde el orden no importa: características, compras, hobbies.
  • <ol> (ordered list) - Usa números. Perfecta para pasos a seguir, rankings, instrucciones donde el orden SÍ importa.
  • <li> (list item) - Cada elemento de la lista, ya sea dentro de ul o ol.
💡 Tip: Puedes meter listas dentro de listas (listas anidadas) para crear subcategorías. Muy útil para menús desplegables.
Tablas

Las tablas organizan información en filas y columnas. Son perfectas para mostrar datos comparativos, horarios, precios, etc.

<table>
  <tr>
    <th>Nombre</th>
    <th>Edad</th>
    <th>Ciudad</th>
  </tr>
  <tr>
    <td>Ana</td>
    <td>25</td>
    <td>Madrid</td>
  </tr>
  <tr>
    <td>Luis</td>
    <td>30</td>
    <td>Barcelona</td>
  </tr>
</table>

Explicación de cada etiqueta:

  • <table> - Crea la tabla. Todo lo que tenga que ver con la tabla va adentro.
  • <tr> - Table Row. Cada <tr> es una fila nueva.
  • <th> - Table Header. Es el encabezado de la columna. Aparece en negrita y centrado por defecto.
  • <td> - Table Data. Cada celda con información dentro de una fila.
✅ Dato: Por cada <tr> (fila), debes tener la misma cantidad de <td> (celdas) para que la tabla se vea ordenada. Si quieres que una celda ocupe varias filas o columnas, existen los atributos rowspan y colspan.
Formularios

Los formularios son la forma en que tu página recibe información del usuario. Desde un simple campo de búsqueda hasta un registro completo.

<form action="/enviar" method="POST">
  <label for="nombre">Tu nombre:</label>
  <input type="text" id="nombre" name="nombre" placeholder="Escribe tu nombre">
  
  <label for="email">Correo:</label>
  <input type="email" id="email" name="email" required>
  
  <label for="mensaje">Mensaje:</label>
  <textarea id="mensaje" name="mensaje" rows="4"></textarea>
  
  <button type="submit">Enviar</button>
</form>

Explicación de cada parte:

  • <form> - El contenedor del formulario. action es a dónde se envía la info, method es cómo se envía (GET o POST).
  • <label> - Texto que describe el campo. El atributo for se conecta con el id del input.
  • <input> - El campo donde el usuario escribe. El atributo type define qué tipo de dato es: text, email, password, number, date, etc.
  • <textarea> - Campo de texto grande, para mensajes largos o comentarios.
  • <button> - Botón que envía el formulario. type="submit" es el que envía los datos.
  • required - Atributo que hace que el campo sea obligatorio. Si el usuario no lo llena, el navegador no deja enviar.
  • placeholder - Texto de ejemplo que aparece dentro del input antes de escribir.
📌 Tipos de input más comunes: text, email, password, number, date, checkbox, radio, file, submit, tel, url, color.
Elementos Semánticos

Antes de HTML5, todo se hacía con <div>. Ahora tenemos etiquetas que describen QUÉ es cada parte de la página. Esto ayuda a los navegadores, a Google y a la accesibilidad.

<header>

Cabecera de la página o de una sección. Aquí suele ir el logo, el título principal y la navegación.

<nav>

Menú de navegación. Contiene los enlaces principales de la página.

<main>

Contenido principal de la página. Debe ser único y no repetirse en otras páginas. Solo debe haber UN <main> por página.

<section>

Agrupa contenido relacionado temáticamente. Por ejemplo: sección de servicios, sección de testimonios.

<article>

Contenido independiente que podría funcionar por sí solo. Ideal para posts de blog, noticias, tarjetas de productos.

<aside>

Contenido relacionado pero no principal. Ejemplo: barra lateral con anuncios, enlaces relacionados, biografía del autor.

<footer>

Pie de página. Aquí va información de contacto, derechos de autor, enlaces a políticas, redes sociales.

✅ Por qué usar etiquetas semánticas:
1. SEO - Google entiende mejor tu contenido y lo posiciona más alto.
2. Accesibilidad - Los lectores de pantalla pueden navegar más fácil.
3. Mantenimiento - Tu código es más fácil de leer para otros desarrolladores y para ti en el futuro.
¡Manos a la obra! Tu primer ejercicio

Ya tienes todas las herramientas básicas. Ahora es momento de construir. Crea una página web simple que contenga:

  • Un <header> con tu nombre como título principal (<h1>)
  • Un <nav> con enlaces ficticios (Inicio, Sobre mí, Contacto)
  • Un <main> con:
    • Un <section> con un párrafo presentándote
    • Un <section> con una lista de tus hobbies (usando <ul>)
    • Un <section> con una imagen tuya o de algo que te guste
  • Un <footer> con un enlace a tu red social favorita

Agrega una tabla con tus 3 películas o videojuegos favoritos (título, año, por qué te gusta). Y asegúrate de que todas las imágenes tengan su atributo alt.
