1. ¿Qué es HTML?

HTML significa HyperText Markup Language. (En español Lenguaje de Marcado de Hipertexto). Este es un lenguaje con el que le dices al navegador mediante etiquetas, qué cosas agregar en tu página web. Cuando entras a una página web, todo lo que ves: títulos, textos, imágenes... eso está escrito en HTML. El navegador (Chrome, Firefox, Edge) lee ese código y lo convierte en lo que tú ves.

HTML no es un lenguaje de programación. No hace cálculos ni toma decisiones. Es un lenguaje de etiquetas. ¿Qué significa eso? Que usas etiquetas para envolver tu contenido.

Para hacer una página web completa, HTML trabaja con dos compañeros. Imagina que estás construyendo una casa:

HTML es la estructura

Son los cimientos, las paredes, el techo. Define dónde va cada cosa. Sin esto, no hay casa.

CSS es el diseño

Es la pintura, los muebles, la decoración. Hace que la página se vea bonita, con colores, formas y que se adapte a cualquier pantalla.

JavaScript es la interacción

Son las luces que se encienden cuando aprietas un interruptor, la puerta que se abre. Hace que la página responda cuando haces clic en algo.

Lo importante: Aprende HTML primero. Es la base. Si no tienes una estructura sólida, el diseño y la interacción no tienen dónde apoyarse. Un buen desarrollador web empieza por aquí.

2. ¿Qué son las etiquetas?

Todo en HTML se escribe con etiquetas. Una etiqueta es simplemente una palabra entre símbolos < y >. Por ejemplo: <h1>.

La mayoría de las etiquetas vienen en pares: una de apertura y una de cierre. La de cierre tiene una barra /. Así:

<h1>Este es mi título</h1>

Todo lo que está entre la apertura y el cierre es el contenido de esa etiqueta. En este caso, "Este es mi título" es el contenido y será tratado como un título principal.

Algunas etiquetas no tienen cierre. Son las etiquetas vacías. Por ejemplo, <br> (salto de línea) o <hr> (línea horizontal). No necesitan envolver nada, simplemente hacen una acción.

Regla de oro: Toda etiqueta que abras, debes cerrarla. Si abres <h1>, más adelante debe ir </h1>. Si abres <p>, luego </p>. Esto evita que el navegador se confunda.
3. La estructura base de todo documento HTML

Toda página web tiene una estructura mínima que debes escribir siempre. No importa si es una página simple o un sitio enorme, esto siempre va al inicio. Es como los cimientos de una casa.

<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8">
        <title>Mi primera página</title>
    </head>
    <body>
        Todo el contenido visible va aquí
    </body>
</html>

Vamos línea por línea, como si fuera la primera vez que ves código:

  • <!DOCTYPE html> — Esta línea no es una etiqueta como tal. Es una declaración. Le dice al navegador: "Oye, esto está escrito en la versión moderna de HTML (HTML5)". Siempre va al inicio del documento. Sin esto, el navegador podría pensar que es una versión vieja y mostrar las cosas mal.
  • <html> — Esta etiqueta envuelve todo el contenido de la página. Es la raíz. Todo lo que escribas debe estar dentro de esta etiqueta. Al final, la cierras con </html>.
  • <head> — Dentro de esta etiqueta va información para el navegador, no para el usuario. Aquí se ponen cosas como el título que aparece en la pestaña, la codificación de caracteres, y enlaces a archivos de estilo. Nada de lo que pongas aquí se ve en la pantalla principal.
  • <meta charset="UTF-8"> — Esta línea define cómo se interpretan los caracteres. UTF-8 es el estándar universal. Permite que se vean bien las tildes, la letra ñ, los emojis y cualquier símbolo especial. Si no pones esto, los acentos pueden salir como caracteres extraños (por ejemplo, "árbol" podría verse como "Ã¡rbol").
  • <title> — Esto es el título que aparece en la pestaña del navegador. No se ve dentro de la página, pero es importante porque el usuario ve esa pestaña. También ayuda a Google a saber de qué trata tu página.
  • </head> — Cierra la cabecera. Todo lo que estaba adentro ya no pertenece al head.
  • <body> — Aquí empieza lo que el usuario ve. Todo el texto, las imágenes, los botones... absolutamente todo lo visible va dentro del body.
  • </body> y </html> — Cierran respectivamente el cuerpo y el documento HTML. Es importante cerrar las etiquetas en el orden inverso al que las abriste.
Nota: Esta estructura no es opcional. Es el punto de partida de cualquier página web. Si la dominas, ya tienes el primer paso.
4. Títulos y párrafos: lo básico del texto

Cuando escribes en HTML, no puedes simplemente poner texto. Debes usar etiquetas que le digan al navegador qué tipo de texto es. Las más importantes son los títulos y los párrafos.

Etiqueta Qué hace Cómo se escribe
<h1> a <h6> Son los títulos y subtítulos. <h1> es el título principal de la página. Solo debe haber uno. <h2> son subtítulos, <h3> sub-subtítulos, y así hasta <h6>. Mientras más alto el número, más pequeño y menos importante es el título. <h1>Bienvenidos a mi web</h1>
<p> Define un párrafo de texto. El navegador automáticamente deja un espacio antes y después de cada párrafo para que se lea mejor. Es la etiqueta que más vas a usar para texto largo. <p>Este es un párrafo normal con información.</p>
<br> Hace un salto de línea. Es una etiqueta que no necesita cierre. Úsala cuando quieras bajar una línea sin crear un párrafo nuevo. Por ejemplo, en direcciones o poemas. Primera línea<br>Segunda línea
<hr> Dibuja una línea horizontal. Sirve para separar secciones de contenido. También es una etiqueta que no necesita cierre. <hr>
Importante: Usar bien los títulos no solo es cuestión de orden. Google lee estas etiquetas para entender tu página. Un <h1> le dice "esto es lo más importante de la página". Los <h2> son temas secundarios. Si los usas bien, tu página puede aparecer más arriba en los resultados de búsqueda.
5. Enlaces: cómo conectar páginas

La palabra "hipertexto" en HTML se refiere a los enlaces. Son los que te permiten ir de una página a otra. Sin enlaces, la web sería un montón de documentos aislados.

<a href="https://www.google.com">Ir a Google</a>

Entendamos esta línea:

  • <a> — Es la etiqueta de enlace. La "a" viene de "anchor" (ancla). Todo lo que pongas entre la apertura y el cierre se vuelve clickeable.
  • href — No es una etiqueta, es un atributo. Los atributos son instrucciones adicionales que se ponen dentro de la etiqueta de apertura. En este caso, href indica a dónde va el enlace. Puede ser una página web externa (como https://google.com) o una página dentro de tu mismo sitio (como "contacto.html").
  • "Ir a Google" — Este es el texto que el usuario ve y donde hace clic.
  • </a> — Cierra el enlace.

Hay un atributo útil que puedes añadir: target="_blank". Si lo pones, el enlace se abre en una nueva pestaña. Así:

<a href="https://www.google.com" target="_blank">Ir a Google (nueva pestaña)</a>
Consejo: Si el enlace lleva a otra página que no es tuya, es buena idea usar target="_blank". Así el usuario no sale de tu sitio. Si el enlace es a una página dentro de tu propio sitio, no lo uses, porque quieres que navegue dentro de la misma pestaña.
6. Imágenes: cómo mostrar fotos

Las imágenes hacen que una página sea visualmente atractiva. Para poner una imagen se usa la etiqueta <img>. Es una etiqueta que no necesita cierre.

<img src="imagenes/foto.jpg" alt="Descripción de la foto">

Explicación:

  • <img> — La etiqueta de imagen. Se abre y se cierra en la misma línea.
  • src — Es un atributo que indica la ubicación de la imagen. Puede ser una ruta dentro de tu proyecto, como "imagenes/foto.jpg", o una dirección de internet, como "https://ejemplo.com/imagen.jpg". Si la imagen está en la misma carpeta que tu archivo HTML, solo pones el nombre del archivo.
  • alt — Es otro atributo. Significa texto alternativo. Si la imagen no se puede cargar, se ve este texto. Además, los lectores de pantalla (que usan personas con discapacidad visual) lo leen. Siempre debes poner alt. Si la imagen es solo decorativa, pon alt="" (vacío).
Ojo con esto: Si no pones el atributo alt, los lectores de pantalla leerán el nombre del archivo (por ejemplo, "foto123.jpg"), lo que no tiene sentido. Es una mala práctica y afecta la accesibilidad de tu página.
7. Listas: para organizar información

Las listas son ideales cuando tienes varios elementos relacionados. Hay dos tipos principales:

<!-- Lista desordenada (con viñetas) -->
<ul>
  <li>Manzanas</li>
  <li>Peras</li>
  <li>Plátanos</li>
</ul>

<!-- Lista ordenada (numerada) -->
<ol>
  <li>Abrir el editor</li>
  <li>Escribir el código</li>
  <li>Guardar el archivo</li>
</ol>

Qué significa cada cosa:

  • <ul> — Significa unordered list (lista desordenada). Usa viñetas (puntos) para cada elemento. Se usa cuando el orden no importa: características de un producto, lista de compras, tus hobbies.
  • <ol> — Significa ordered list (lista ordenada). Usa números. Se usa cuando el orden sí importa: pasos de una receta, instrucciones, un ranking.
  • <li> — Significa list item (elemento de la lista). Cada elemento de la lista va entre <li> y </li>. Esto aplica tanto para ul como para ol.
Tip: Puedes poner una lista dentro de otra. Por ejemplo, un elemento de la lista puede tener una sublista. Esto se llama listas anidadas y es muy útil para crear menús o categorías.
8. Tablas: información en filas y columnas

Las tablas se usan para mostrar información que tiene estructura de filas y columnas. Por ejemplo: horarios, precios, comparativas.

<table>
  <tr>
    <th>Nombre</th>
    <th>Edad</th>
    <th>Ciudad</th>
  </tr>
  <tr>
    <td>Ana</td>
    <td>25</td>
    <td>Madrid</td>
  </tr>
  <tr>
    <td>Luis</td>
    <td>30</td>
    <td>Barcelona</td>
  </tr>
</table>

Partes de una tabla:

  • <table> — La etiqueta que crea la tabla. Todo lo que tenga que ver con la tabla va adentro.
  • <tr> — Significa table row (fila de tabla). Cada <tr> es una fila nueva.
  • <th> — Significa table header (encabezado de tabla). Define el título de una columna. Por defecto se ve en negrita y centrado.
  • <td> — Significa table data (dato de tabla). Es una celda normal con información.

Cómo se organiza: Primero defines los encabezados con <th> dentro de una fila. Luego, por cada fila de datos, pones tantos <td> como columnas tengas. Si la primera fila tiene 3 <th>, todas las filas deben tener 3 <td> para que la tabla quede ordenada.

Dato útil: Si quieres que una celda ocupe varias columnas, puedes usar el atributo colspan. Por ejemplo: <td colspan="2"> hace que esa celda ocupe el espacio de dos columnas.
9. Etiquetas semánticas: darle significado a tu página

Antes de HTML5, las páginas se armaban con una etiqueta genérica llamada <div>. No había forma de saber qué parte era el menú, cuál el contenido principal, cuál el pie de página. Ahora tenemos etiquetas semánticas, que le dicen al navegador y a Google qué es cada parte de tu página.

<header>

La cabecera de la página. Aquí va el título principal, el logo y normalmente el menú de navegación. Es como la entrada de una casa.

<nav>

El menú de navegación. Contiene los enlaces principales de la página. Aquí es donde el usuario elige a qué sección ir.

<main>

El contenido principal de la página. Solo puede haber uno. Aquí va lo más importante, lo que hace única a esta página.

<section>

Agrupa contenido relacionado. Por ejemplo: una sección de servicios, una sección de testimonios, una sección de contacto.

<article>

Un contenido que podría funcionar por sí solo. Por ejemplo: un post de blog, una noticia, un comentario. Es como un artículo de periódico.

<aside>

Contenido relacionado pero no principal. Normalmente una barra lateral con información extra, anuncios, o enlaces relacionados.

<footer>

El pie de página. Aquí va información de contacto, derechos de autor, enlaces a políticas de privacidad, etc.

¿Por qué usar estas etiquetas? Por tres razones: 1) Google entiende mejor tu página y la posiciona más alto. 2) Los lectores de pantalla pueden navegar más fácil (accesibilidad). 3) Tu código es más fácil de leer y mantener.
Manos a la obra: tu primer proyecto

Ya viste las etiquetas más importantes. Ahora es momento de construir. Vas a crear una página web que incluya lo siguiente. Escribe esto en tu editor de código (puede ser el Bloc de notas, Visual Studio Code, o cualquier editor) y guárdalo como index.html. Luego ábrelo con tu navegador.

Requisitos de tu primera página:

  • Un <header> con tu nombre como título principal (<h1>).
  • Un <nav> con tres enlaces: Inicio, Sobre mí, Contacto. Por ahora, pon el atributo href="#" (el # significa que es un enlace vacío que no lleva a ningún lado, pero sirve para practicar).
  • Un <main> que contenga:
    • Una <section> con un párrafo presentándote. Cuenta quién eres y por qué quieres aprender a programar.
    • Una <section> con una lista de tus hobbies. Usa <ul> para poner viñetas.
    • Una <section> con una imagen. Puede ser una foto tuya, un dibujo, o cualquier imagen que tengas. Acuérdate de poner el atributo alt con una descripción de la imagen.
  • Un <footer> con un enlace a tu red social favorita (si no tienes, pon un enlace a Google o a cualquier página).
Reto extra si quieres practicar más: Agrega una tabla con tus 3 películas o videojuegos favoritos. La tabla debe tener columnas para: Título, Año, y por qué te gusta. Usa <th> para los encabezados.
