1. Colores en HTML

En HTML (y CSS) puedes poner colores de varias formas. Los colores se usan en atributos como color, bgcolor, background, etc. Hay 4 formas principales de escribir un color:

🎨 1. Nombres de colores

Los más fáciles de recordar. HTML reconoce 140 nombres de colores.

red blue green yellow orange purple pink cyan 
<font color="red">Texto rojo</font>
<body bgcolor="lightblue">...</body>
🎨 2. RGB (Red, Green, Blue)

Define la cantidad de rojo, verde y azul. Cada valor va de 0 a 255.

rgb(255,0,0) = rojo
rgb(0,255,0) = verde
rgb(0,0,255) = azul
rgb(255,255,0) = amarillo
rgb(255,0,255) = magenta
rgb(0,255,255) = cian
<font color="rgb(255,100,50)">Texto naranja</font>
🎨 3. HEX (Hexadecimal)

Es la forma más común. Empieza con # y luego 6 dígitos (0-9 y A-F).

#FF0000 = rojo
#00FF00 = verde
#0000FF = azul
#FFFF00 = amarillo
#FF00FF = magenta
#00FFFF = cian
#000000 = negro
#FFFFFF = blanco
#c89a4c = dorado
<font color="#c89a4c">Texto dorado</font>
🎨 4. HSL (Hue, Saturation, Lightness)

Define el color por matiz (0-360), saturación (0-100%) y luminosidad (0-100%).

hsl(0,100%,50%) = rojo
hsl(120,100%,50%) = verde
hsl(240,100%,50%) = azul
hsl(60,100%,50%) = amarillo
<font color="hsl(30,100%,50%)">Texto naranja</font>
💡 Consejo: Para páginas web modernas, lo más usado es HEX (#RRGGBB) y RGB/RGBA. RGBA es como RGB pero con un canal alpha para transparencia: rgba(255,100,50,0.5).
2. SVG: Gráficos Vectoriales

SVG (Scalable Vector Graphics) es una forma de dibujar gráficos directamente en HTML. A diferencia de las imágenes normales (jpg, png), los SVG no pierden calidad al agrandarse. Son perfectos para logos, iconos e ilustraciones.

<!-- Círculo -->
<svg width="100" height="100">
  <circle cx="50" cy="50" r="40" 
          fill="#e6c38a" stroke="#c89a4c" stroke-width="3"/>
</svg>

<!-- Rectángulo con bordes redondeados -->
<svg width="150" height="80">
  <rect x="10" y="10" width="130" height="60" 
        fill="#c97e3a" rx="15"/>
</svg>

<!-- Línea -->
<svg width="200" height="50">
  <line x1="10" y1="25" x2="190" y2="25" 
        stroke="#fff" stroke-width="3"/>
</svg>

<!-- Estrella con polygon -->
<svg width="100" height="100">
  <polygon points="50,10 61,35 90,35 68,52 79,80 50,63 21,80 32,52 10,35 39,35" 
           fill="#e6c38a" />
</svg>

Formas básicas de SVG:

  • <circle> - Círculo: cx, cy (centro), r (radio)
  • <rect> - Rectángulo: x, y, width, height, rx (bordes redondeados)
  • <line> - Línea: x1,y1 (inicio) x2,y2 (fin)
  • <polygon> - Polígono: points (coordenadas separadas por espacios)
  • <path> - Forma personalizada (más avanzado)
  • <text> - Texto dentro del SVG
<!-- Texto dentro de SVG -->
<svg width="200" height="50">
  <text x="20" y="35" fill="#e6c38a" font-size="20">Hola SVG!</text>
</svg>
🖼️ ¿Para qué sirve SVG?
Logos que se ven bien en cualquier tamaño, iconos que puedes cambiar de color con CSS, gráficos simples, animaciones básicas, mapas interactivos.
3. Iframes: Páginas dentro de páginas

Un <iframe> es una "ventana" que muestra otra página web dentro de la tuya. Es súper útil para incrustar videos de YouTube, mapas de Google, widgets de redes sociales, etc.

<!-- Iframe básico -->
<iframe src="https://ejemplo.com" width="100%" height="300"></iframe>

<!-- Video de YouTube -->
<iframe width="560" height="315" 
  src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
  frameborder="0" 
  allowfullscreen>
</iframe>

<!-- Mapa de Google Maps -->
<iframe 
  src="https://www.google.com/maps/embed?pb=!1m18!1m12!1m3!1d3021.123456!2d-0.123456!3d40.123456!2m3!1f0!2f0!3f0!3m2!1i1024!2i768!4f13.1!3m3!1m2!1s0x0%3A0x0!2zNDDCsDA3JzI0LjQiTiAwwrAwNyc1Ny40Ilc!5e0!3m2!1ses!2ses!4v1234567890" 
  width="600" height="450" 
  style="border:0;" 
  allowfullscreen="" 
  loading="lazy">
</iframe>

Atributos importantes del iframe:

  • src - La URL de lo que quieres mostrar.
  • width / height - Tamaño en píxeles o porcentaje.
  • frameborder - Si muestra borde (0 = sin borde).
  • allowfullscreen - Permite pantalla completa.
  • loading="lazy" - Carga solo cuando el usuario se acerca (mejora rendimiento).
  • sandbox - Restricciones de seguridad para el contenido incrustado.
⚠️ Dónde conseguir código de iframe:
YouTube: Compartir → Insertar → Copias el código.
Google Maps: Compartir → Insertar un mapa → Copias el código.
4. Audio y Video (Repaso)

Repaso de multimedia nativa con más detalles sobre atributos útiles:

<!-- Audio con controles -->
<audio controls>
  <source src="cancion.mp3" type="audio/mpeg">
  <source src="cancion.ogg" type="audio/ogg">
  Tu navegador no soporta audio.
</audio>

<!-- Video con poster (imagen de vista previa) -->
<video width="640" height="360" controls poster="vista-previa.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Tu navegador no soporta video.
</video>

<!-- Video en bucle y silenciado (útil para fondos) -->
<video autoplay loop muted>
  <source src="fondo.mp4" type="video/mp4">
</video>

Atributos de audio y video:

  • controls - Muestra botones de reproducción.
  • autoplay - Empieza automáticamente (en muchos navegadores requiere muted).
  • loop - Se repite en bucle.
  • muted - Empieza silenciado.
  • poster - (Solo video) Imagen que se ve antes de reproducir.
  • preload - Carga el video antes de reproducir (auto, metadata, none).
✅ Formatos recomendados:
Audio: MP3 (universal), OGG (alternativo)
Video: MP4 (H.264) es el más compatible, WebM es alternativa libre.
5. Metadatos Avanzados (SEO)

Los metadatos van dentro del <head> y le dan información a los navegadores y a Google sobre tu página.

<head>
  <!-- Metadatos básicos -->
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  
  <!-- SEO: Descripción que aparece en Google -->
  <meta name="description" content="Aprende HTML desde cero con ejemplos prácticos y manual completo">
  
  <!-- SEO: Palabras clave (menos importante hoy en día) -->
  <meta name="keywords" content="HTML, tutorial, web, aprender, manual">
  
  <!-- Autor de la página -->
  <meta name="author" content="Kerolmi">
  
  <!-- Control de robots de Google -->
  <meta name="robots" content="index, follow">
  
  <!-- Redes sociales (Open Graph) -->
  <meta property="og:title" content="Manual de HTML - Kerolmi">
  <meta property="og:description" content="Aprende HTML fácil y rápido">
  <meta property="og:image" content="https://tusitio.com/imagen.jpg">
  <meta property="og:url" content="https://tusitio.com/html.html">
  <meta property="og:type" content="website">
  
  <!-- Twitter Cards -->
  <meta name="twitter:card" content="summary_large_image">
  <meta name="twitter:title" content="Manual de HTML - Kerolmi">
  <meta name="twitter:description" content="Aprende HTML fácil y rápido">
  <meta name="twitter:image" content="https://tusitio.com/imagen.jpg">
  
  <title>Kerolmi · Manual de HTML</title>
</head>
📈 ¿Por qué son importantes?
Meta description - Es el texto que aparece debajo del título en Google. ¡Influye en que la gente haga clic!
Open Graph - Controla cómo se ve tu página cuando la compartes en Facebook, WhatsApp, Twitter.
6. Favicon y Link

El favicon es ese pequeño ícono que aparece en la pestaña del navegador. También puedes usar la etiqueta <link> para enlazar otros recursos.

<!-- Favicon básico (icono en la pestaña) -->
<link rel="icon" type="image/x-icon" href="favicon.ico">
<!-- O para PNG -->
<link rel="icon" type="image/png" href="favicon.png">

<!-- Tamaños diferentes para dispositivos -->
<link rel="apple-touch-icon" sizes="180x180" href="apple-touch-icon.png">
<link rel="icon" type="image/png" sizes="32x32" href="favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="favicon-16x16.png">

<!-- Enlazar CSS externo -->
<link rel="stylesheet" href="estilos.css">

<!-- Enlazar fuentes de Google Fonts -->
<link href="https://fonts.googleapis.com/css2?family=Roboto&display=swap" rel="stylesheet">

Usos de <link>:

  • rel="icon" - Favicon de la página.
  • rel="stylesheet" - Archivo CSS externo.
  • rel="preconnect" - Mejora rendimiento para recursos externos.
  • rel="canonical" - URL principal (evita contenido duplicado).
  • rel="alternate" - Versiones alternativas (idiomas, RSS).
✅ Tip: Los favicon se ven en la pestaña, en marcadores, y en la pantalla de inicio de móviles. Dale identidad visual a tu página con un buen icono.
🚀 Proyecto Final: Tu Primera Página Completa

¡Felicidades! Has llegado al final de los fundamentos básicos de HTML. Ahora es momento de crear una página que integre TODO lo que aprendiste en las 3 partes:

📋 Requisitos del proyecto:
  • Estructura semántica - <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>
  • Enlaces internos - Menú que navegue dentro de la misma página usando #id
  • Enlaces externos - Que abran en nueva pestaña (target="_blank")
  • Listas - Al menos una lista ordenada y una desordenada
  • Tabla - Con información de horarios, precios, o tus cosas favoritas
  • Formulario - Con diferentes tipos de input (text, email, date, color, range)
  • Imágenes - Con atributo alt obligatorio
  • SVG o Canvas - Dibuja algo (un logo, un círculo, una estrella)
  • Iframe - Incrusta un video de YouTube o un mapa de Google Maps
  • Colores - Usa HEX, RGB y nombres de colores en diferentes elementos
  • Favicon - Ponle un icono a tu página
  • Meta descripción - Para que Google muestre un buen resumen
  • Responsive - Usa la meta etiqueta viewport
🎯 Reto extra:
Crea un menú desplegable usando solo HTML (sin JavaScript). Pista: puedes usar <select> y <option> o <details> y <summary>.
