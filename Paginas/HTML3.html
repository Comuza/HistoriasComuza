HTML desde Cero · Parte 3

Ya sabes escribir HTML. Sabes qué es una etiqueta, un atributo, cómo hacer formularios, cómo poner videos, cómo guardar datos. Pero ahora viene lo importante: cómo trabajar como un profesional sin gastar un solo peso. Porque hay mucha gente que cree que necesita comprar programas caros o pagar hosting para empezar. Y no es así. En esta parte te voy a mostrar las herramientas gratuitas que usan los desarrolladores de verdad, cómo organizar tu código, cómo poner tu página en internet sin pagar, y las buenas prácticas que te van a ahorrar años de dolores de cabeza.

1. Herramientas gratuitas para programar (sin pagar nada)

Para hacer páginas web solo necesitas dos cosas: un editor de código (donde escribes el HTML) y un navegador (para ver lo que hiciste). Ambas son gratis.

Visual Studio Code (VS Code)

Es el editor que usan la mayoría de los desarrolladores profesionales. Es gratis, liviano y tiene herramientas que te ayudan a escribir más rápido. Lo puedes descargar de code.visualstudio.com. Funciona en Windows, Mac y Linux.

Google Chrome / Firefox

Cualquier navegador moderno sirve. Pero Chrome y Firefox tienen herramientas para desarrolladores (presiona F12) que te permiten inspeccionar tu código, ver errores y probar cosas en tiempo real.

Git y GitHub

Git es un programa que guarda el historial de tus cambios (como un "control de versiones"). GitHub es una página donde puedes subir tu código gratis. Es como un respaldo en la nube y también sirve para mostrar tu trabajo.

Imágenes gratis

No necesitas pagar por fotos. Páginas como Unsplash, Pexels, Pixabay tienen imágenes gratis que puedes usar en tus proyectos. Solo asegúrate de leer los términos de uso.

¿Y el bloc de notas? Técnicamente puedes escribir HTML en el bloc de notas. Pero es como querer construir una casa con un martillo de juguete. VS Code te ayuda con colores (para que veas qué es una etiqueta y qué es texto), te sugiere etiquetas mientras escribes, y te avisa si te equivocaste. Vale la pena instalarlo.
2. Cómo usar VS Code (lo básico que necesitas saber)

Cuando abres VS Code por primera vez, parece complicado. Pero solo necesitas saber unas pocas cosas para empezar.

Crear un archivo nuevo:

  • Archivo → Nuevo archivo (o Ctrl + N).
  • Escribe tu código HTML.
  • Archivo → Guardar (o Ctrl + S). Para escogerle el nombre, si o si debes darle este nombre "index.html" sino, jamas cargara la pagina, porque es el nombre por defecto que busca el navegador.

Abrir una carpeta (recomendado):

  • Es mejor tener una carpeta para cada proyecto. Archivo → Abrir carpeta. Elige o crea una carpeta donde vas a guardar tus archivos.
  • En el panel izquierdo ves todos tus archivos. Puedes crear nuevos archivos haciendo clic en los íconos.

Extensiones útiles (son gratis):

  • Live Server — Instálala. Te permite ver tu página en el navegador y que se actualice sola cada vez que guardas cambios. Haces clic derecho en tu archivo HTML y eliges "Open with Live Server".
  • Prettier — Ordena tu código automáticamente. Le pones un atajo (Alt + Shift + F) y te limpia las indentaciones.
  • Auto Rename Tag — Cuando cambias el nombre de una etiqueta de apertura, cambia automáticamente la de cierre. 
Atajos de teclado que te van a salvar la vida:
- Ctrl + S : Guardar
- Ctrl + Z : Deshacer
- Ctrl + Shift + Z : Rehacer
- Ctrl + F : Buscar dentro del archivo
- Ctrl + D : Selecciona la siguiente coincidencia (útil para cambiar varias palabras iguales)
- Ctrl + / : Comenta o descomenta una línea
Tip: Si escribes ! (solo el signo de exclamación) y presionas Enter en un archivo HTML, VS Code te genera automáticamente la estructura base del documento. Es un atajo que te ahorra tiempo.
3. Ver tu página en el navegador

Hay varias formas de ver lo que estás construyendo. Todas son gratis.

Opción 1: Abrir el archivo directamente

  • Ve a la carpeta donde guardaste tu archivo .html.
  • Haz doble clic en el archivo. Se abrirá en tu navegador.
  • Si haces cambios, guarda y actualiza la página (F5) para verlos.

Opción 2: Live Server (la mejor)

  • Instala la extensión Live Server en VS Code.
  • Haz clic derecho en tu archivo HTML y elige "Open with Live Server".
  • Se abre una pestaña en el navegador. Cada vez que guardas (Ctrl+S), la página se actualiza sola. No tienes que recargar manualmente.

Opción 3: Ver desde el teléfono

  • Si estás usando Live Server, fíjate en la dirección que te aparece (algo como http://127.0.0.1:5500).
  • Tu computadora y tu teléfono deben estar en la misma red WiFi.
  • En el teléfono, abre el navegador y escribe la dirección de tu computadora pero con la IP. Ejemplo: http://192.168.1.10:5500.
  • Así puedes ver cómo se ve tu página en móvil mientras la desarrollas.
Herramienta secreta: Chrome DevTools
Presiona F12 en Chrome. Se abre un panel donde puedes ver el código de cualquier página, inspeccionar elementos, probar estilos en vivo, ver errores, y simular cómo se ve en diferentes tamaños de pantalla. Es la herramienta más poderosa que tienes. Aprende a usarla y te vas a convertir en otro nivel.
4. Cómo organizar tus archivos (para no volverte loco)

Cuando empiezas, es fácil tener todos los archivos en el mismo lugar. Pero cuando tu proyecto crece, se vuelve un caos. Por eso los desarrolladores organizan las carpetas desde el principio. 

mi-proyecto/
│
├── index.html          (la página principal)
├── acerca.html         (otra página)
├── contacto.html       (otra página)
│
├── css/
│   └── estilo.css      (todos los estilos van aquí)
│
├── js/
│   └── script.js       (todo JavaScript va aquí)
│
├── img/
│   ├── foto1.jpg       (todas las imágenes)
│   ├── foto2.png
│   └── icono.svg
│
├── fonts/
│   └── (fuentes personalizadas si usas)
│
└── videos/             (si tienes videos propios)
    └── presentacion.mp4

Reglas simples:

  • La página principal SIEMPRE se llama index.html. Es la que el navegador busca por defecto.
  • Los archivos de CSS (estilos) van en una carpeta llamada css/.
  • Los archivos de JavaScript van en una carpeta llamada js/.
  • Las imágenes van en img/ o images/.
  • No uses espacios en los nombres de archivos. Usa guiones: mi-foto.jpg en lugar de mi foto.jpg. Los espacios pueden dar problemas.
  • Usa minúsculas para los nombres de archivos. En algunos servidores, Imagen.JPG y imagen.jpg son diferentes. 
<!-- Cómo enlazar archivos cuando están en carpetas -->

<!-- CSS (dentro de la carpeta css) -->
<link rel="stylesheet" href="css/estilo.css">

<!-- JavaScript (dentro de la carpeta js) -->
<script src="js/script.js"></script>

<!-- Imagen (dentro de la carpeta img) -->
<img src="img/foto.jpg" alt="descripción">

<!-- Enlace a otra página que está en la misma carpeta -->
<a href="acerca.html">Acerca de</a>

<!-- Enlace a una página que está en una subcarpeta -->
<a href="productos/catalogo.html">Catálogo</a>
5. Subir tu página a internet sin pagar

Tener tu página en tu computadora está bien, pero lo bonito es que otros la vean. Hay formas gratuitas de subir tu página a internet. La más popular es GitHub Pages.

GitHub Pages: tu sitio web gratis para siempre

  • Paso 1: Crea una cuenta en github.com (es gratis).
  • Paso 2: Crea un "repositorio" (es como una carpeta en la nube). Dale un nombre, por ejemplo: mi-primera-web. Marca la opción "Public".
  • Paso 3: Sube tus archivos. Puedes arrastrar tu carpeta con los archivos HTML, CSS, imágenes, etc.
  • Paso 4: Ve a la configuración del repositorio (pestaña "Settings"). Busca la sección "Pages". En "Branch", selecciona "main" y guarda.
  • Paso 5: En unos minutos, tu página estará disponible en https://tuusuario.github.io/nombre-del-repositorio/.
Alternativas gratis:
  • Netlify — Arrastras tu carpeta y te dan un enlace. Es más simple que GitHub Pages.
  • Vercel — Muy fácil también.
  • Cloudflare Pages — Otra opción gratuita.
Todas tienen planes gratis que son más que suficientes para empezar.
¿Y si quiero un dominio .com? Puedes comprar un dominio por unos pocos dólares al año (en lugares como Namecheap, GoDaddy, o Cloudflare). Luego lo conectas a tu página de GitHub Pages o Netlify. Pero para empezar, el dominio que te dan gratis está perfecto.
6. Buenas prácticas para escribir HTML

Escribir código que funciona es el primer paso. Escribir código que otros puedan leer y mantener es lo que separa a un principiante de un profesional. Estas son las reglas que siguen los que saben.

1. Indentación (los espacios)

  • Cada vez que abres una etiqueta dentro de otra, aumentas la indentación (dos espacios o un tabulador).
  • Esto hace que el código sea legible. Si todo está pegado a la izquierda, es un infierno. 
<!-- MAL (todo pegado) -->
<body>
<header>
<h1>Título</h1>
</header>
<main>
<p>Texto</p>
</main>
</body>

<!-- BIEN (con indentación) -->
<body>
  <header>
    <h1>Título</h1>
  </header>
  <main>
    <p>Texto</p>
  </main>
</body>

2. Comentarios

  • Puedes dejar notas en tu código que el navegador ignora. Sirven para recordarte qué hace cada parte.
  • Se escriben así: <!-- esto es un comentario -->.

3. Nombres claros para id y class

  • Usa nombres que digan qué es, no cómo se ve. class="texto-rojo" es mala idea porque si después quieres que sea azul, el nombre queda mal. Mejor class="aviso-importante".
  • Usa guiones para separar palabras: id="seccion-contacto" en lugar de id="seccioncontacto".

4. Cerrar todas las etiquetas

  • No olvides cerrar las etiquetas que abriste. Un error común es dejar sin cerrar un <div> o <section>. Después todo se desordena.

5. Usar etiquetas semánticas

  • Ya lo vimos en la Parte 1. Usa <header>, <nav>, <main>, <section>, <article>, <footer>. No uses <div> para todo.
Validar tu código: Antes de subir tu página a internet, pásala por validator.w3.org. Es una página que revisa si tu HTML está bien escrito. Te dice los errores y advertencias. Es gratis y es lo que usan los profesionales para asegurarse de que no hay errores tontos.
7. Cómo seguir aprendiendo (sin pagar cursos)

Ya sabes lo básico de HTML. Ahora viene la parte más importante: aprender a aprender. Porque nadie se vuelve programador solo con un manual. Es un camino de práctica constante.

Dónde buscar ayuda cuando te trabes:

  • MDN Web Docs (developer.mozilla.org) — Es la documentación oficial. Está en español. Si tienes una duda sobre una etiqueta, busca ahí. Es más confiable que cualquier otra fuente.
  • Stack Overflow — Es un foro donde los programadores preguntan y responden. Seguro alguien ya tuvo tu mismo problema. Aprende a buscar bien.
  • YouTube — Hay miles de tutoriales gratis. Busca canales como "Midudev", "FalconMasters", "Leonidas Esteban". Aprendes viendo cómo trabajan otros.

Cómo practicar:

  • Clona páginas que te gusten — Elige una página simple e intenta copiarla. No mires el código original, hazlo de memoria. Cuando te trabes, revisa cómo lo hicieron.
  • Haz proyectos personales — No esperes a "saber suficiente". Desde ahora, haz una página de tus hobbies, un portafolio, una página para un familiar. Aprender haciendo es lo único que funciona.
  • Mantén un repositorio en GitHub — Sube todo lo que hagas. Sirve como respaldo y como portafolio para cuando busques trabajo.
La mentalidad correcta:
  • No esperes entender todo a la primera. Los programadores pasan el 80% del tiempo buscando en Google.
  • No te compares con los que llevan 10 años. Compárate con cómo estabas ayer.
  • Hacer algo que funciona, aunque sea feo, es mejor que no hacer nada. Después lo mejoras.
  • Los errores son parte del trabajo. Cada error que resuelves es un aprendizaje que nadie te puede quitar.
Proyecto final: tu primera página web profesional

Has llegado hasta aquí. Ya sabes escribir HTML, sabes usar atributos, multimedia, formularios, sabes organizar archivos, usar VS Code, y subir tu página a internet gratis. Ahora toca poner todo junto.

Tu misión final:

  • Crea una página web personal sobre ti. Puede ser un portafolio, una página de tus intereses, lo que quieras.
  • Organiza tus archivos en carpetas: css/, js/, img/.
  • Usa etiquetas semánticas (header, nav, main, section, footer).
  • Incluye al menos una imagen con su alt.
  • Incluye un enlace interno (que lleve a otra sección de la misma página).
  • Incluye un enlace externo (que lleve a otra página, como tu GitHub o redes).
  • Usa un formulario con al menos dos tipos de input (puede ser de contacto).
  • Incluye un video o audio (puede ser incrustado de YouTube).
  • Sube tu proyecto a GitHub Pages o Netlify para que cualquiera lo vea.
Bonus (si quieres destacar):
  • Haz que tu página se vea bien en móvil (usa la meta viewport que aprendiste en la Parte 1).
  • Valida tu HTML en validator.w3.org y corrige los errores.
  • Pon una meta description en el head para que Google muestre un buen resumen.
  • Guarda en LocalStorage el nombre del usuario si lo escribe, y salúdalo cuando vuelva.
Resumen de herramientas gratuitas que viste:
  • VS Code — Editor de código.
  • Live Server — Ver cambios en tiempo real.
  • GitHub Pages / Netlify — Subir tu página gratis.
  • Unsplash / Pexels — Imágenes gratis.
  • Google Fonts — Fuentes gratis para tus textos.
  • Font Awesome — Íconos gratis para tus botones y menús.
  • validator.w3.org — Revisar que tu código esté bien.
  • Chrome DevTools (F12) — Inspeccionar y probar cambios en vivo.
