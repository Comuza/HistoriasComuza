1. Atributos Globales

Los atributos son configuraciones extra que le pones a las etiquetas (id, clases etc.). Estos atributos funcionan en TODAS las etiquetas. Esto se les denominan "atributos globales". Esta es la base para darle comportamiento e identidad a tus elementos.

id

La id es un identificador ÚNICO para cada elemento.

<h1 id="titulo-principal">Bienvenidos</h1>
class

Identificador REUTILIZABLE. Puedes poner la misma clase a muchos elementos. Sirve para darles el mismo estilo con CSS o el mismo comportamiento con JavaScript.

<p class="texto-destacado">Este texto se ve especial</p>
title

Añade un tooltip (texto que aparece al pasar el mouse). Muy útil para dar información extra sin ocupar espacio.

<abbr title="HyperText Markup Language">HTML</abbr>
data-*

Atributos personalizados. Puedes inventar tus propios atributos usando "data-" al inicio. Muy útil para guardar información que luego JavaScript puede leer.

<button data-id="123" data-color="rojo">Botón</button>
💡 Diferencia clave entre id y class: id es como tu número de identificación (único), class es como el uniforme del colegio (muchos lo pueden usar). Usa id para elementos únicos, class para grupos.
2. Enlaces Internos y Anclas

Ya viste cómo hacer enlaces a otras páginas. Pero también puedes hacer enlaces que lleven a otra parte de la MISMA página. Esto se hace con anclas (id) y enlaces especiales.

<!-- Primero, creas un destino con id -->
<h2 id="seccion-contacto">Contáctanos</h2>

<!-- Luego, haces un enlace que lleve ahí -->
<a href="#seccion-contacto">Ir a Contacto</a>

<!-- También puedes enlazar a una sección de otra página -->
<a href="pagina.html#seccion-contacto">Ir a Contacto en otra página</a>

Cómo funciona:

  • El # en el href indica que es un enlace interno.
  • Después del # pones el id del elemento al que quieres llegar.
  • Cuando alguien hace clic, el navegador se desplaza suavemente (o de golpe) hasta ese elemento.
  • Esto es súper útil para menús largos, FAQs, o páginas con mucho contenido.
<!-- Ejemplo práctico: menú de navegación interno -->
<nav>
  <a href="#inicio">Inicio</a>
  <a href="#servicios">Servicios</a>
  <a href="#contacto">Contacto</a>
</nav>

<section id="inicio">...</section>
<section id="servicios">...</section>
<section id="contacto">...</section>
✅ Tip pro: Si quieres un enlace que no haga nada (para pruebas), usa href="javascript:void(0)" o simplemente href="#". El # solo recarga la página al inicio, así que ojo con eso.
3. Audio y Video (Multimedia Nativa)

Antes de HTML5, para poner un video necesitabas Flash (un plugin externo). Ahora HTML tiene etiquetas propias para audio y video. Funcionan en todos los navegadores modernos.

<!-- Audio simple -->
<audio controls>
  <source src="cancion.mp3" type="audio/mpeg">
  <source src="cancion.ogg" type="audio/ogg">
  Tu navegador no soporta audio.
</audio>

<!-- Video simple -->
<video width="640" height="360" controls>
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Tu navegador no soporta video.
</video>

Atributos importantes:

  • controls - Muestra los botones de reproducción, pausa, volumen.
  • autoplay - El video/audio empieza automáticamente (cuidado, muchos navegadores lo bloquean si no hay interacción).
  • loop - Se repite en bucle.
  • muted - Empieza silenciado (útil para autoplay).
  • poster - (Solo video) Imagen que se muestra antes de reproducir.
<!-- Video con más opciones -->
<video width="100%" controls autoplay muted loop poster="imagen-preview.jpg">
  <source src="video.mp4" type="video/mp4">
  Tu navegador no soporta video.
</video>
🎵 Formatos soportados:
Audio: MP3, WAV, OGG
Video: MP4 (H.264), WebM, OGG
Tip: Incluye varios formatos con <source> para compatibilidad máxima.
4. Iframes: Contenido de otras páginas

Un iframe es como una "ventana" dentro de tu página que muestra contenido de otra web. Sirve para incrustar videos de YouTube, mapas de Google, widgets de redes sociales, etc.

<!-- Incrustar un video de YouTube -->
<iframe width="560" height="315" 
  src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
  frameborder="0" 
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" 
  allowfullscreen>
</iframe>

<!-- Incrustar un mapa de Google Maps -->
<iframe 
  src="https://www.google.com/maps/embed?pb=!1m18!1m12!1m3!1d..." 
  width="600" height="450" 
  style="border:0;" 
  allowfullscreen="" 
  loading="lazy">
</iframe>

Atributos importantes del iframe:

  • src - La URL de lo que quieres mostrar (YouTube, Google Maps, etc.)
  • width / height - Tamaño del iframe en píxeles o porcentaje.
  • allowfullscreen - Permite que el contenido se ponga en pantalla completa.
  • loading="lazy" - Carga el contenido solo cuando el usuario se acerca (mejora el rendimiento).
  • sandbox - Aplica restricciones de seguridad al contenido incrustado.
⚠️ Seguridad: No todos los sitios permiten ser incrustados en iframes (tienen protección). YouTube, Vimeo, Google Maps sí permiten. Si quieres incrustar algo, busca la opción "Compartir" → "Insertar" para obtener el código correcto.
5. Formularios Avanzados

HTML5 trajo nuevos tipos de input y atributos que facilitan mucho la creación de formularios. Validan datos automáticamente y mejoran la experiencia en móviles.

<form>
  <!-- Input de email con validación automática -->
  <input type="email" placeholder="tu@email.com" required>
  
  <!-- Input de número con mínimo y máximo -->
  <input type="number" min="1" max="100" step="1">
  
  <!-- Input de fecha (calendario nativo) -->
  <input type="date">
  
  <!-- Input de color (selector de color) -->
  <input type="color">
  
  <!-- Input de rango (slider) -->
  <input type="range" min="0" max="100">
  
  <!-- Input de teléfono (muestra teclado numérico en móvil) -->
  <input type="tel">
  
  <!-- Input de búsqueda (con botón para borrar) -->
  <input type="search">
  
  <!-- Input de URL (valida que sea una dirección web) -->
  <input type="url">
  
  <!-- Datalist (autocompletado personalizado) -->
  <input list="ciudades">
  <datalist id="ciudades">
    <option>Madrid</option>
    <option>Barcelona</option>
    <option>Valencia</option>
  </datalist>
</form>

Atributos nuevos de validación:

  • required - Campo obligatorio.
  • min / max - Valor mínimo y máximo (para number, date, range).
  • pattern - Expresión regular para validar el formato (ej: patrón para código postal).
  • placeholder - Texto de ejemplo dentro del input.
  • autocomplete - Sugiere valores guardados por el navegador (on/off).
📱 Diferencia en móviles: Al usar type="tel" aparece el teclado numérico. Con type="email" aparece el teclado con @ y .com. Con type="date" aparece un selector de fecha nativo. ¡Mejora muchísimo la experiencia del usuario!
6. APIs de HTML5 (Geolocalización y Almacenamiento)

HTML5 no solo es etiquetas. También trajo APIs (interfaces de programación) que permiten hacer cosas increíbles con JavaScript. Aquí las dos más útiles:

Geolocalización

Obtiene la ubicación del usuario (con su permiso). Útil para tiendas cercanas, clima, mapas, etc.

navigator.geolocation.getCurrentPosition(function(posicion) {
  console.log("Latitud: " + posicion.coords.latitude);
  console.log("Longitud: " + posicion.coords.longitude);
});
LocalStorage y SessionStorage

Guardan datos en el navegador del usuario. LocalStorage guarda para siempre (hasta que se borre manualmente), SessionStorage guarda hasta que se cierra la pestaña.

// Guardar datos
localStorage.setItem("nombre", "Juan");
localStorage.setItem("tema", "oscuro");

// Leer datos
let nombre = localStorage.getItem("nombre");

// Borrar datos
localStorage.removeItem("nombre");
localStorage.clear(); // Borra todo
✅ ¿Para qué sirve LocalStorage?
Guardar preferencias del usuario (tema claro/oscuro), recordar qué productos puso en el carrito aunque cierre la página, guardar la sesión sin necesidad de servidor, etc. Es como una mini base de datos en el navegador.
7. SEO y Buenas Prácticas

Escribir HTML no es solo hacer que funcione. También hay que pensar en Google (para que te encuentren) y en las personas con discapacidad (accesibilidad).

📌 Estructura correcta de títulos

Solo UN <h1> por página. Luego <h2> para secciones, <h3> para subsecciones. No saltes niveles (no pongas un <h4> después de un <h2> sin un <h3> de por medio).

📌 Atributo alt en imágenes

Siempre, siempre, siempre. Describe qué hay en la imagen. Si es decorativa, pon alt="" (vacío). Esto ayuda a personas con discapacidad visual y a Google.

📌 Etiquetas semánticas

Usa <header>, <nav>, <main>, <section>, <article>, <aside>, <footer>. No uses <div> para todo. Google entiende mejor la estructura de tu página con etiquetas semánticas.

📌 Meta descripción

Agrega esto en el <head>: <meta name="description" content="Descripción de tu página que aparece en Google">. Es el texto que ven los usuarios en los resultados de búsqueda.

<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Aprende HTML desde cero con ejemplos prácticos">
  <meta name="keywords" content="HTML, tutorial, web, aprender">
  <meta name="author" content="Kerolmi">
  <title>Mi página - Aprende HTML</title>
</head>
✅ Resumen de buenas prácticas:
1. Usa etiquetas semánticas.
2. Solo un <h1> por página.
3. Siempre <alt> en imágenes.
4. Meta description para que Google muestre un buen resumen.
5. Código indentado y ordenado (legible para humanos y máquinas).
