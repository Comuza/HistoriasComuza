1. Atributos: cómo darle instrucciones extra a las etiquetas

Hasta ahora has visto etiquetas como <h1> o <p>. Pero las etiquetas pueden llevar instrucciones adicionales. Esas instrucciones se llaman atributos. Los atributos siempre van dentro de la etiqueta de apertura, y tienen esta forma: nombre="valor".

<etiqueta atributo="valor">contenido</etiqueta>

Por ejemplo, cuando hiciste enlaces, usaste el atributo href para decir a dónde ir. Así:

<a href="https://google.com">Ir a Google</a>

Aquí href es un atributo, y "https://google.com" es su valor.

Hay atributos que funcionan en cualquier etiqueta. Se llaman atributos globales. Los más importantes son:

Atributo Qué hace Ejemplo
id Le pone un nombre único a una etiqueta. Como un DNI. Solo una etiqueta puede tener ese id. Sirve para que luego podamos referirnos a ella desde enlaces internos o desde CSS. <h1 id="titulo-principal">Hola</h1>
class Le pone una o varias etiquetas de grupo. A diferencia del id, muchas etiquetas pueden compartir la misma clase. Sirve para que todas se vean igual con CSS o tengan el mismo comportamiento. <p class="texto-importante">Aviso</p>
title Añade un texto que aparece al pasar el mouse por encima. Como una ayuda emergente. <abbr title="Lenguaje de Marcado">HTML</abbr>
style Permite poner colores y diseños directamente en la etiqueta (aunque lo normal es usar CSS aparte). <p style="color: red;">Texto rojo</p>
Diferencia clave entre id y class: Piensa en el id como tu número de identificación personal (es único, solo tuyo). La class es como el uniforme del colegio (muchas personas pueden usarlo). Usa id para elementos únicos, class para grupos.
2. Enlaces internos: saltar dentro de la misma página

Ya sabes hacer enlaces que llevan a otra página. Pero también puedes hacer enlaces que lleven a otra parte de la misma página. Esto es muy útil cuando tienes una página larga con muchas secciones, o una página de preguntas frecuentes.

Se hace en dos pasos:

  • Paso 1: Le pones un id al lugar al que quieres llegar. Por ejemplo, si quieres que un enlace lleve a la sección de contacto, le pones id="contacto" a esa sección.
  • Paso 2: El enlace, en lugar de poner una dirección web, pone # seguido del id. Por ejemplo: href="#contacto".
<!-- Primero, creas un destino con id -->
<h2 id="seccion-contacto">Contáctanos</h2>
<p>Aquí va el formulario de contacto...</p>

<!-- Luego, en otro lugar de la página, haces un enlace que lleva ahí -->
<a href="#seccion-contacto">Ir a la sección de contacto</a>

<!-- También puedes enlazar a una sección de otra página -->
<a href="contacto.html#formulario">Ir al formulario en otra página</a>

El # es la clave. Le dice al navegador: "esto no es una página nueva, es un lugar dentro de esta misma página". Cuando alguien hace clic, el navegador se desplaza suavemente (o de golpe) hasta el elemento con ese id.

<!-- Ejemplo práctico: un menú que lleva a diferentes partes -->
<nav>
  <a href="#inicio">Inicio</a>
  <a href="#servicios">Servicios</a>
  <a href="#contacto">Contacto</a>
</nav>

<section id="inicio">
  <h2>Bienvenido</h2>
</section>

<section id="servicios">
  <h2>Nuestros servicios</h2>
</section>

<section id="contacto">
  <h2>Escríbenos</h2>
</section>
Tip: Si quieres un enlace que no haga nada (por ejemplo, para probar algo), puedes poner href="#". Pero ojo: ese enlace llevará al inicio de la página..
3. Audio y Video: poner música y películas sin plugins

Antes de HTML5, para poner un video en una página tenías que usar un programa aparte llamado Flash. Era complicado, lento y a veces no funcionaba. Ahora HTML tiene sus propias etiquetas para audio y video. Funcionan en todos los navegadores modernos, a estas etiquetas, se les denomina "Etiquetas de audio".

Etiqueta de audio:

<audio controls>
  <source src="cancion.mp3" type="audio/mpeg">
  <source src="cancion.ogg" type="audio/ogg">
  Tu navegador no soporta audio.
</audio>

Etiqueta de video:

<video width="640px" height="360px" controls>
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Tu navegador no soporta video.
</video>

Atributos que puedes usar (van dentro de la etiqueta de apertura):

  • controls — Muestra los botones de play, pausa, volumen. Sin esto, el usuario no puede controlar el audio/video.
  • autoplay — Empieza a reproducirse automáticamente cuando carga la página. (Ojo: muchos navegadores lo bloquean si no hay interacción del usuario).
  • loop — Cuando termina, vuelve a empezar. Se repite en bucle.
  • muted — Empieza silenciado. Es útil si quieres usar autoplay, porque los navegadores suelen permitir autoplay solo si está silenciado.
  • poster — (Solo para video) Es una imagen que se muestra antes de que empiece el video, como una portada.
  • width — Le dice al navegador el ancho del video en píxeles, el numero en "". son los px
  • height — Le dice al navegador la altura del video en píxeles, el numero en "" son los px.
<!-- Video con poster y autoplay silenciado -->
<video width="100%" controls autoplay muted loop poster="portada.jpg">
  <source src="video.mp4" type="video/mp4">
  Tu navegador no soporta video.
</video>
¿Por qué poner varios <source>? No todos los navegadores soportan los mismos formatos. MP4 funciona en casi todos, pero si pones también un archivo WebM, cubres más casos. El navegador elige el primero que puede reproducir.
4. Iframes: mostrar una página dentro de otra página

Un iframe es como una ventana dentro de tu página que muestra contenido de otra página web. Sirve para poner videos de YouTube, mapas de Google Maps, widgets de redes sociales, o cualquier otra cosa que quieras incrustar.

La etiqueta es <iframe>. El atributo más importante es src, que dice qué página mostrar.

<!-- Incrustar un video de YouTube -->
<iframe width="560" height="315" 
  src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
  allowfullscreen>
</iframe>

<!-- Incrustar un mapa de Google Maps -->
<iframe 
  src="https://www.google.com/maps/embed?pb=!1m18... " 
  width="600" height="450" 
  allowfullscreen 
  loading="lazy">
</iframe>

Atributos comunes en iframes:

  • src — La dirección de la página que quieres mostrar. Puede ser un video de YouTube, un mapa, una página tuya, etc.
  • width y height — El tamaño del iframe en píxeles. También puedes usar porcentajes como width="100%".
  • allowfullscreen — Permite que el contenido (como un video de YouTube) se ponga en pantalla completa.
  • loading="lazy" — Hace que el iframe se cargue solo cuando el usuario se acerca a él. Mejora la velocidad de la página.
Ojo con esto: No todas las páginas permiten ser incrustadas en iframes. Algunas tienen protección para que no las pongan dentro de otra página. YouTube, Vimeo, Google Maps sí lo permiten. Para incrustar algo, busca la opción "Compartir" → "Insertar" o "Integrar". Ellos te dan el código listo para copiar. 
<!-- Cómo se ve un iframe responsivo (que se ajusta al ancho) -->
<iframe 
  src="https://www.youtube.com/embed/dQw4w9WgXcQ" 
  style="width: 100%; max-width: 560px; height: 315px;" 
  allowfullscreen>
</iframe>
  • max El max es la cantidad maxima de px que puede alcanzar.
  • min El min es la cantidad minima de px que puede alcanzar.
    • 5. Formularios más inteligentes: nuevos tipos de input

    En la Parte 1 viste los formularios básicos: texto, email, botón. HTML5 trajo muchos tipos nuevos de input que hacen la vida más fácil. Cada tipo tiene su propia validación y en el teléfono muestra el teclado adecuado.

    <form>
  <!-- Email: valida que tenga @ y dominio -->
  <input type="email" placeholder="tu@email.com" required>
  
  <!-- Número: solo acepta números, puedes poner mínimo y máximo -->
  <input type="number" min="1" max="100" step="1">
  
  <!-- Teléfono: en móvil muestra teclado numérico -->
  <input type="tel" placeholder="+34 600 000 000">
  
  <!-- Fecha: aparece un calendario para elegir -->
  <input type="date">
  
  <!-- Color: muestra un selector de colores -->
  <input type="color">
  
  <!-- Rango: un slider que puedes deslizar -->
  <input type="range" min="0" max="100" value="50">
  
  <!-- Búsqueda: tiene un botón para borrar lo escrito -->
  <input type="search" placeholder="Buscar...">
  
  <!-- URL: valida que sea una dirección web -->
  <input type="url" placeholder="https://ejemplo.com">
  
  <!-- Datalist: autocompletado con opciones sugeridas -->
  <input list="ciudades" placeholder="Escribe tu ciudad">
  <datalist id="ciudades">
    <option>Madrid</option>
    <option>Barcelona</option>
    <option>Valencia</option>
    <option>Sevilla</option>
  </datalist>
  
  <button type="submit">Enviar</button>
</form>

    Atributos nuevos para validar:

    • required — El campo es obligatorio. Si no se llena, el navegador no deja enviar.
    • min / max — Valor mínimo y máximo (para number, date, range).
    • step — De cuánto en cuánto aumenta o disminuye (para number y range).
    • pattern — Una "fórmula" para validar el formato. Por ejemplo, para un código postal de España: pattern="[0-9]{5}" (cinco números).
    • placeholder — Texto de ejemplo que aparece dentro del campo antes de escribir.
    • autocomplete — Si pones "off", el navegador no sugerirá datos guardados. Útil para campos de contraseña.
    Ventajas de estos tipos nuevos: En el móvil, type="tel" muestra el teclado numérico. type="email" muestra el teclado con @ y .com. type="date" muestra un calendario nativo. Todo esto mejora muchísimo la experiencia del usuario sin que tú tengas que programar nada.
    6. Guardar datos en el navegador: LocalStorage

    HTML5 trajo una herramienta muy útil: la posibilidad de guardar información en el navegador del usuario. Se llama LocalStorage. Es como una pequeña base de datos que vive en el ordenador o teléfono de quien visita tu página.

    ¿Para qué sirve?

    • Guardar preferencias del usuario (tema claro u oscuro, idioma, tamaño de letra).
    • Recordar lo que el usuario puso en el carrito de compras, aunque cierre la página.
    • Guardar el nombre del usuario para saludarlo cuando vuelva.
    • Recordar si ya vio un mensaje para no mostrarlo otra vez.

    Para usar LocalStorage necesitas un poco de JavaScript, pero es muy sencillo. Aquí van los comandos básicos:

    <!-- Guardar un dato -->
<script>
  localStorage.setItem("nombre", "Juan");
  localStorage.setItem("tema", "oscuro");
</script>

<!-- Leer un dato -->
<script>
  let nombreGuardado = localStorage.getItem("nombre");
  console.log(nombreGuardado); // Muestra "Juan"
</script>

<!-- Borrar un dato específico -->
<script>
  localStorage.removeItem("nombre");
</script>

<!-- Borrar todos los datos guardados -->
<script>
  localStorage.clear();
</script>

    ¿Cómo se usa en la práctica? Por ejemplo, quieres que cuando el usuario escriba su nombre en un campo, ese nombre se guarde y aparezca la próxima vez que entre a la página:

    <input type="text" id="nombreUsuario" placeholder="Tu nombre">
<button onclick="guardarNombre()">Guardar</button>
<p id="saludo"></p>

<script>
  // Cuando la página carga, ver si ya hay un nombre guardado
  let nombre = localStorage.getItem("nombre");
  if(nombre) {
    document.getElementById("saludo").innerHTML = "Hola " + nombre + "! Bienvenido de vuelta.";
  }
  
  function guardarNombre() {
    let nombre = document.getElementById("nombreUsuario").value;
    localStorage.setItem("nombre", nombre);
    alert("Nombre guardado!");
  }
</script>
    Dato importante: Los datos de LocalStorage se quedan guardados hasta que se borren manualmente. Si el usuario cierra el navegador y vuelve a abrirlo, los datos siguen ahí. Si quieres que los datos se borren cuando se cierra la pestaña, existe SessionStorage (funciona igual pero se borra al cerrar).
    7. SEO: que Google te encuentre

    SEO significa "Search Engine Optimization" (optimización para motores de búsqueda). Es el conjunto de buenas prácticas para que Google muestre tu página arriba cuando alguien busca algo relacionado con lo que haces.

    Cosas que puedes hacer desde HTML para mejorar tu SEO:

    • Un solo <h1> por página — El título principal. Google lo interpreta como "esto es lo más importante de esta página".
    • Jerarquía de títulos — Después del <h1>, usa <h2> para secciones, <h3> para subsecciones. No saltes niveles (no pongas un <h4> después de un <h2> sin un <h3>).
    • Atributo alt en todas las imágenes — Describe qué hay en la imagen. Google no "ve" imágenes, solo lee el texto alternativo. Además, es obligatorio para accesibilidad.
    • Etiquetas semánticas — Usa <header>, <nav>, <main>, <section>, <article>, <footer>. Google entiende mejor la estructura de tu página.
    • Meta descripción — Es un texto que aparece en los resultados de búsqueda debajo del título. Atrae clics.
    <head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Aprende HTML desde cero con ejemplos prácticos. Explicaciones sencillas para principiantes.">
  <meta name="keywords" content="HTML, tutorial, aprender HTML, programación web">
  <meta name="author" content="Kerolmi">
  <title>Aprende HTML desde Cero - Tutorial Completo</title>
</head>
    Resumen de buenas prácticas HTML:
    1. Un solo <h1> por página.
    2. Usa <alt> en todas las imágenes.
    3. Usa etiquetas semánticas (header, nav, main, section, article, footer).
    4. Pon meta description en el head.
    5. Escribe un título de página (title) claro y descriptivo.
    6. Código indentado y ordenado (fácil de leer para humanos y máquinas).

    Crea una página que incluya:

    • Un menú con enlaces internos (usando id) que lleven a diferentes secciones de la misma página. Mínimo 3 secciones.

    • Un audio con controles (puede ser un archivo de prueba, o usa un audio de ejemplo de internet).

    • Al menos un elemento con id y otro con class.
    Reto extra (si quieres practicar más):
    Usa LocalStorage con JavaScript. Crea un input donde el usuario pueda escribir su nombre, un botón para guardarlo, y muestra un saludo con el nombre guardado cuando la página cargue. Si el nombre ya está guardado, que aparezca el saludo sin necesidad de escribir otra vez.
